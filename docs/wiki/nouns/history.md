# HISTORY

The stage that records the supersession chain: when a claim replaces an old one, the old one is never deleted but retained in the claim log marked superseded. Renders the live tip, terse breadcrumbs for user mind-changes, and keeps superseded agent claims in the log for audit without auto-rendering them.
