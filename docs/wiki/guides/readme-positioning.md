# Readme Positioning

## Why this exists

The founding problem is the scarcity of perspective. This project is a beginners' Python Hello World, and the README and docs are meant to capture a friendly, welcoming tone. Every human touchpoint is valuable not merely for epistemic hygiene or auditability, but because it brings human taste and a unique perspective, in the Vervaeke relevance-realization sense: the particular way you see this thing.

<!-- citations: [^105d3-4] [^82403-8] -->
## Thesis

Knowledge should be held faithful to the person — faithful to the words you chose, because they carry a perspective worth more than any restatement. A lossy summary does not merely drop a qualifier; it strips the perspective underneath — the particular way you see this thing. The differentiator is domain-neutral: how knowledge is held, not which domain holds it. Claude Code is just the current host integration. <!-- [^105d3-5] -->

## Status

This project is research-grade, not a product you should trust with anything you cannot afford to re-derive. Local-first is a trust detail, not a headline pitch. <!-- [^105d3-6] -->

## Where this is going

The following capabilities are spec-only and under active development — not yet shipped: citation-anchored capture, integrity-by-construction, and live decay/contradiction reasoning. They are presented here to signal direction, not as present-tense fact. <!-- [^105d3-7] -->
