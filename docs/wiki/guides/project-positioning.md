# Project Positioning and Philosophy

## Lead Positioning

The README's lead positioning is that the project captures human discernment as the scarce, appreciating asset, not the model's fluent output. Verbatim-cited injection is framed as proof of the stance that human perspective is treasured, not as the headline differentiator. The thesis line is 'provenance over fluency,' landing on fidelity to the person's chosen words as carrying a perspective worth more than any restatement. <!-- [^105d3-95ff2] -->

(Previously: the project was anchored on 'memory' as a category word.) The project is no longer anchored on 'memory' as a category word because it was identified as a weak differentiator in a crowded market. <!-- [^105d3-480c6] -->

The project is not scoped to coding agents specifically; the verbatim-cited knowledge approach is domain-neutral and Claude Code is just the current host integration. <!-- [^105d3-77611] -->

## Founding Bias

The README's founding-bias framing is that a human correction is the one input a model cannot generate for itself — the human's discernment of what matters — rather than a cost asymmetry of tokens vs. attention. The README weaves a Vervaeke-adjacent spirit — discernment as relevance realization, perspectival knowing as the irreplaceable human deposit — implicitly through the capture and show-your-work sections without name-dropping. <!-- [^105d3-88099] -->

## README Sections

The README carries a 'Why a wiki, and not a summary' section framed as anchor-and-grow vs. dump-and-dilute, presenting the rejected PRODUCT_MODEL.md as an explicit architectural stance. The 'What it refuses to do' section enumerates trust-earning refusals: distill-don't-dump, evidence-with-decay, contradictions preserved, librarian-not-author. The Status section is honest: research-grade, not a product to trust with anything you cannot afford to re-derive. <!-- [^105d3-8e93f] -->

## Wiki Mission

The wiki's mission is to explain why the code looks the way it does, including the archaeology of how user decisions evolved. It is a desired-state spec plus that provenance — not a raw event log, but curated current state with load-bearing history. <!-- [^26c90-c2b2b] -->
