# PC

PC is proactive-context: it captures durable project direction from agent transcripts, stores current truth plus history as cited artifacts, and injects the relevant slice back into the agent before it acts. Read [how PC works](../../docs/product-spec/how-it-works.md) for the landed architecture, and read [the deep audit](../../docs/product-spec/deep-audit-2026-07-02.md) when compiled PC artifacts look incomplete or incorrect so you can identify the concrete failure mode and fix the generating path or rerun the proper PC flow instead of hand-patching derived output.
